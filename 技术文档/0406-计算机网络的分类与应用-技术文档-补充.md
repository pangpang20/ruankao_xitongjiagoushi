- **设计复杂**: 需要仔细规划

- **昂贵**: 更多设备和基础设施
- **需要专业知识**: 需要熟练人员管理

**应用:**
- 大型企业网络
- 大学校园
- 医院网络
- 多建筑企业园区
- 数据中心（spine-leaf架构）
- ISP网络

**最佳实践:**
- 使用分层设计（思科三层模型）
- 在关键层实施冗余
- 尽可能标准化设备
- 彻底记录拓扑
- 规划增长和可扩展性

## 4. Classification by Transmission Media | 按传输介质分类

### 4.1 Wired Networks | 有线网络

**English:**

**1. Twisted Pair Cable:**

**Unshielded Twisted Pair (UTP):**
- Most common LAN cabling
- Categories: Cat5e, Cat6, Cat6a, Cat7, Cat8
- RJ-45 connector
- Maximum distance: 100 meters

| Category | Max Speed      | Bandwidth | Applications       |
| -------- | -------------- | --------- | ------------------ |
| Cat5e    | 1 Gbps         | 100 MHz   | Gigabit Ethernet   |
| Cat6     | 10 Gbps (55m)  | 250 MHz   | 10G short distance |
| Cat6a    | 10 Gbps (100m) | 500 MHz   | 10GBASE-T          |
| Cat7     | 10 Gbps        | 600 MHz   | Data centers       |
| Cat8     | 40 Gbps        | 2000 MHz  | Server connections |

**Shielded Twisted Pair (STP):**
- Additional shielding for EMI protection
- Used in high-interference environments
- More expensive than UTP

**2. Coaxial Cable:**
- Central conductor + insulation + braided shield + outer jacket
- Types: RG-6 (Cable TV, Internet), RG-59 (CCTV)
- Higher bandwidth than twisted pair
- Better EMI immunity
- Applications: Cable Internet, CATV, legacy Ethernet

**3. Fiber Optic Cable:**

**Single-Mode Fiber (SMF):**
- Core diameter: 8-10 µm
- Uses laser light source
- Long distance: 10-100+ km
- Higher cost
- Applications: Long-distance telecom, WAN, data center interconnects

**Multi-Mode Fiber (MMF):**
- Core diameter: 50 or 62.5 µm
- Uses LED light source
- Short distance: 300m-2km
- Lower cost than SMF
- Applications: LANs, building backbone, data centers

**Advantages of Fiber:**
- Extremely high bandwidth (Tbps capable)
- Immune to electromagnetic interference
- Long distance without signal degradation
- Secure (difficult to tap)
- Lightweight and thin
- No electrical hazard

**Disadvantages of Fiber:**
- Higher cost (cable, transceivers, installation)
- Fragile (requires careful handling)
- Specialized skills for installation
- More expensive to repair

**屏蔽双绞线（STP）:**

- 中心导体 + 绝缘 + 编织屏蔽 + 外护套

- 带宽高于双绞线

- 应用: 有线互联网、有线电视、传统以太网

**3. 光纤:**

**单模光纤（SMF）:**
- 芯径: 8-10 µm
- 使用激光光源
- 长距离: 10-100+公里
- 成本更高
- 应用: 长途电信、WAN、数据中心互联

**多模光纤（MMF）:**
- 芯径: 50或62.5 µm
- 使用LED光源
- 短距离: 300米-2公里
- 成本低于SMF
- 应用: LAN、建筑骨干网、数据中心

**光纤优势:**
- 极高带宽（Tbps能力）
- 不受电磁干扰
- 长距离无信号衰减
- 安全（难以窃听）
- 轻巧纤细
- 无电气危险

**光纤劣势:**
- 成本更高（电缆、收发器、安装）
- 易碎（需要小心处理）
- 安装需要专业技能
- 维修更昂贵

**4. Satellite Communication:**

- Low power, low data rate (0.3-50 kbps)
- Sub-GHz bands (868 MHz EU, 915 MHz US)
- Applications: Smart cities, agriculture, asset tracking

**Wireless Advantages:**
- Mobility and flexibility
- Easy deployment (no cabling)
- Scalable (easy to add devices)
- Cost-effective for temporary setups
- Reaches difficult areas

**Wireless Disadvantages:**
- Lower bandwidth than wired
- Security vulnerabilities (encryption required)
- Interference issues
- Limited range
- Power consumption (for battery devices)
- Weather-dependent (some technologies)

## 5. Classification by Network Architecture | 按网络架构分类

### 5.1 Client-Server Architecture | 客户端-服务器架构

**English:**

**Definition:**
Client-server architecture is a computing model where client devices request services and resources from centralized servers.

**Components:**
- **Clients**: End-user devices (PCs, smartphones, thin clients)
- **Servers**: Powerful computers providing services
- **Network**: Communication infrastructure

**Server Types:**
1. **File Server**: Centralized file storage and sharing
2. **Database Server**: Data management (SQL Server, Oracle, MySQL)
3. **Web Server**: HTTP services (Apache, Nginx, IIS)
4. **Application Server**: Business logic (Java EE, .NET)
5. **Email Server**: Email services (Exchange, Postfix)
6. **Print Server**: Printer management
7. **Authentication Server**: User authentication (Active Directory, LDAP)

**Characteristics:**
- Centralized control and management
- Client initiates requests, server responds
- Servers always running and available
- Multiple clients per server
- Specialized servers for different services

**Advantages:**
- **Centralized Management**: Easy to administer and backup
- **Security**: Centralized access control and authentication
- **Scalability**: Add more servers as needed
- **Resource Sharing**: Efficient use of resources
- **Reliability**: Professional server hardware
- **Performance**: Powerful servers handle heavy processing
- **Consistency**: Single source of truth for data

**Disadvantages:**
- **Single Point of Failure**: Server downtime affects all clients
- **Cost**: Expensive server hardware and software licenses
- **Network Dependency**: Requires reliable network
- **Scalability Limits**: Server capacity constraints
- **Complexity**: Requires IT expertise
- **Bandwidth**: Heavy network traffic

**Architectural Patterns:**

**1. Two-Tier (Client-Server):**
```
[Client] ←→ [Database Server]
```
- Direct database access from client
- Simple but security concerns
- Legacy applications

**2. Three-Tier:**
```
[Client] ←→ [Application Server] ←→ [Database Server]
    (Presentation)   (Business Logic)      (Data)
```
- Separation of concerns
- Better security
- Modern web applications

**3. N-Tier:**
```
[Client] ←→ [Web Server] ←→ [App Server] ←→ [DB Server]
         ←→ [Cache Server] ←→ [Integration Services]
```
- Highly scalable
- Microservices architecture
- Cloud-native applications

**Applications:**
- Enterprise applications (ERP, CRM)
- Email systems
- Database applications
- Web applications
- File sharing systems
- Corporate intranets

**中文:**

**定义:**
客户端-服务器架构是一种计算模型，其中客户端设备从集中式服务器请求服务和资源。

**组件:**
- **客户端**: 最终用户设备（PC、智能手机、瘦客户端）
- **服务器**: 提供服务的强大计算机
- **网络**: 通信基础设施

**服务器类型:**
1. **文件服务器**: 集中式文件存储和共享
2. **数据库服务器**: 数据管理（SQL Server、Oracle、MySQL）
3. **Web服务器**: HTTP服务（Apache、Nginx、IIS）
4. **应用服务器**: 业务逻辑（Java EE、.NET）
5. **邮件服务器**: 电子邮件服务（Exchange、Postfix）
6. **打印服务器**: 打印机管理
7. **认证服务器**: 用户认证（Active Directory、LDAP）

**特征:**
- 集中控制和管理
- 客户端发起请求，服务器响应
- 服务器始终运行和可用
- 每个服务器多个客户端
- 不同服务的专用服务器

**优点:**
- **集中管理**: 易于管理和备份
- **安全**: 集中访问控制和认证
- **可扩展性**: 根据需要添加更多服务器
- **资源共享**: 资源的高效使用
- **可靠性**: 专业服务器硬件
- **性能**: 强大的服务器处理繁重处理
- **一致性**: 数据的单一真实来源

**缺点:**
- **单点故障**: 服务器停机影响所有客户端
- **成本**: 昂贵的服务器硬件和软件许可证
- **网络依赖**: 需要可靠的网络
- **可扩展性限制**: 服务器容量限制
- **复杂性**: 需要IT专业知识
- **带宽**: 繁重的网络流量

**架构模式:**

**1. 两层（客户端-服务器）:**
```
[客户端] ←→ [数据库服务器]
```
- 客户端直接访问数据库
- 简单但有安全问题
- 传统应用

**2. 三层:**
```
[客户端] ←→ [应用服务器] ←→ [数据库服务器]
 （表示层）     （业务逻辑）        （数据）
```
- 关注点分离
- 更好的安全性
- 现代Web应用

**3. N层:**
```
[客户端] ←→ [Web服务器] ←→ [应用服务器] ←→ [数据库服务器]
         ←→ [缓存服务器] ←→ [集成服务]
```
- 高度可扩展
- 微服务架构
- 云原生应用

**应用:**
- 企业应用（ERP、CRM）
- 电子邮件系统
- 数据库应用
- Web应用
- 文件共享系统
- 企业内网

### 5.2 Peer-to-Peer (P2P) Architecture | 点对点架构

**English:**

**Definition:**
Peer-to-peer architecture is a distributed network where all nodes (peers) have equal capabilities and responsibilities, acting as both clients and servers.

**Characteristics:**
- Decentralized architecture
- No dedicated servers
- Each peer can request and provide services
- Dynamic topology (peers join/leave)
- Self-organizing network

**Types of P2P:**

**1. Pure P2P (Unstructured):**
- No central server
- Peers communicate directly
- Examples: Early Gnutella, Bitcoin
- Advantages: No single point of failure, high resilience
- Disadvantages: Difficult to locate resources, scalability issues

**2. Hybrid P2P:**
- Central index/tracker server
- Data transfer peer-to-peer
- Examples: Napster, early BitTorrent
- Advantages: Faster resource location, better performance
- Disadvantages: Central server dependency

**3. Structured P2P (DHT):**
- Distributed Hash Table (DHT)
- Organized topology (Chord, Kademlia)
- Examples: BitTorrent (DHT mode), IPFS
- Advantages: Efficient resource location, scalable
- Disadvantages: More complex to implement

**Advantages:**
- **Decentralization**: No single point of failure
- **Cost-Effective**: No expensive server infrastructure
- **Scalability**: Resources scale with peers
- **Resource Utilization**: Leverages all peer resources
- **Resilience**: Network survives peer failures
- **Censorship Resistance**: Difficult to shut down

**Disadvantages:**
- **Security Risks**: Direct peer exposure
- **Inconsistency**: No central data authority
- **Performance**: Variable quality based on peers
- **Management**: Difficult to control and monitor
- **Legal Issues**: Copyright concerns (file sharing)
- **Bandwidth**: Unequal contribution from peers

**Applications:**
1. **File Sharing**: BitTorrent, eMule, IPFS
2. **Cryptocurrency**: Bitcoin, Ethereum (blockchain networks)
3. **Communication**: Skype (early versions), Jitsi
4. **Content Delivery**: IPFS, WebTorrent
5. **Collaborative Computing**: BOINC, Folding@home
6. **Gaming**: Some multiplayer games
7. **Messaging**: Matrix, Session

**P2P vs. Client-Server:**

| Feature      | P2P                | Client-Server     |
| ------------ | ------------------ | ----------------- |
| Architecture | Decentralized      | Centralized       |
| Cost         | Low                | High              |
| Scalability  | High               | Limited by server |
| Management   | Difficult          | Easy              |
| Security     | Challenging        | Controlled        |
| Performance  | Variable           | Consistent        |
| Reliability  | High (distributed) | Depends on server |

**中文:**

**定义:**
点对点架构是一种分布式网络，其中所有节点（对等点）具有相同的能力和责任，既充当客户端又充当服务器。

**特征:**
- 去中心化架构
- 无专用服务器
- 每个对等点都可以请求和提供服务
- 动态拓扑（对等点加入/离开）
- 自组织网络

**P2P类型:**

**1. 纯P2P（无结构）:**
- 无中央服务器
- 对等点直接通信
- 示例: 早期Gnutella、比特币
- 优点: 无单点故障、高弹性
- 缺点: 难以定位资源、可扩展性问题

**2. 混合P2P:**
- 中央索引/跟踪服务器
- 点对点数据传输
- 示例: Napster、早期BitTorrent
- 优点: 更快的资源定位、更好的性能
- 缺点: 中央服务器依赖

**3. 结构化P2P（DHT）:**
- 分布式哈希表（DHT）
- 有组织的拓扑（Chord、Kademlia）
- 示例: BitTorrent（DHT模式）、IPFS
- 优点: 高效的资源定位、可扩展
- 缺点: 实现更复杂

**优点:**
- **去中心化**: 无单点故障
- **成本效益**: 无需昂贵的服务器基础设施
- **可扩展性**: 资源随对等点扩展
- **资源利用**: 利用所有对等点资源
- **弹性**: 网络在对等点故障时继续运行
- **抗审查**: 难以关闭

**缺点:**
- **安全风险**: 直接对等点暴露
- **不一致性**: 无中央数据权威
- **性能**: 基于对等点的可变质量
- **管理**: 难以控制和监控
- **法律问题**: 版权问题（文件共享）
- **带宽**: 对等点的不平等贡献

**应用:**
1. **文件共享**: BitTorrent、eMule、IPFS
2. **加密货币**: 比特币、以太坊（区块链网络）
3. **通信**: Skype（早期版本）、Jitsi
4. **内容分发**: IPFS、WebTorrent
5. **协作计算**: BOINC、Folding@home
6. **游戏**: 一些多人游戏
7. **消息传递**: Matrix、Session

**P2P vs. 客户端-服务器:**

| 特性     | P2P          | 客户端-服务器 |
| -------- | ------------ | ------------- |
| 架构     | 去中心化     | 集中化        |
| 成本     | 低           | 高            |
| 可扩展性 | 高           | 受服务器限制  |
| 管理     | 困难         | 容易          |
| 安全     | 具有挑战性   | 受控          |
| 性能     | 可变         | 一致          |
| 可靠性   | 高（分布式） | 取决于服务器  |

### 5.3 Cloud-Based Architecture | 基于云的架构

**English:**

**Definition:**
Cloud-based architecture delivers computing services over the Internet, providing on-demand access to shared pools of configurable resources.

**Cloud Service Models:**

**1. Infrastructure as a Service (IaaS):**
- Virtualized computing resources
- Examples: AWS EC2, Azure VMs, Google Compute Engine
- User manages: OS, middleware, applications
- Provider manages: Servers, storage, networking, virtualization

**2. Platform as a Service (PaaS):**
- Development and deployment platforms
- Examples: AWS Elastic Beanstalk, Google App Engine, Heroku
- User manages: Applications and data
- Provider manages: Runtime, middleware, OS, infrastructure

**3. Software as a Service (SaaS):**
- Complete applications over the Internet
- Examples: Office 365, Salesforce, Gmail, Dropbox
- User manages: Only data and usage
- Provider manages: Everything else

**Cloud Deployment Models:**

**1. Public Cloud:**
- Services available to general public
- Multi-tenant architecture
- Examples: AWS, Azure, Google Cloud
- Advantages: Low cost, high scalability, no maintenance
- Disadvantages: Less control, security concerns, shared resources

**2. Private Cloud:**
- Dedicated infrastructure for single organization
- On-premises or hosted
- Examples: VMware vCloud, OpenStack private deployments
- Advantages: Full control, enhanced security, compliance
- Disadvantages: High cost, requires management, limited scalability

**3. Hybrid Cloud:**
- Combination of public and private
- Data/workload portability
- Examples: Azure Arc, AWS Outposts
- Advantages: Flexibility, cost optimization, gradual migration
- Disadvantages: Complex management, integration challenges

**4. Multi-Cloud:**
- Using multiple cloud providers
- Avoid vendor lock-in
- Best-of-breed services
- Advantages: Redundancy, flexibility, negotiation power
- Disadvantages: Increased complexity, skills requirements

**Cloud Characteristics:**
- **On-Demand Self-Service**: Provision resources automatically
- **Broad Network Access**: Accessible via standard protocols
- **Resource Pooling**: Multi-tenant model
- **Rapid Elasticity**: Scale up/down quickly
- **Measured Service**: Pay-per-use billing

**Advantages:**
- **Scalability**: Elastic resource scaling
- **Cost Efficiency**: Pay-as-you-go, no capital expenditure
- **Accessibility**: Access from anywhere
- **Disaster Recovery**: Built-in redundancy and backup
- **Automatic Updates**: Provider handles maintenance
- **Global Reach**: Data centers worldwide
- **Innovation**: Latest technologies available

**Disadvantages:**
- **Internet Dependency**: Requires reliable connectivity
- **Security and Privacy**: Data stored off-premises
- **Compliance**: Regulatory concerns
- **Vendor Lock-In**: Difficult to migrate
- **Limited Control**: Provider manages infrastructure
- **Downtime Risk**: Provider outages affect service

**Applications:**
- Web and mobile applications
- Big data analytics
- Artificial Intelligence/Machine Learning
- Content delivery and streaming
- Collaboration tools
- Backup and disaster recovery
- Development and testing environments

**中文:**

**定义:**
基于云的架构通过互联网提供计算服务，提供对可配置资源共享池的按需访问。

**云服务模型:**

**1. 基础设施即服务（IaaS）:**
- 虚拟化计算资源
- 示例: AWS EC2、Azure VM、Google Compute Engine
- 用户管理: 操作系统、中间件、应用

- 用户管理: 应用程序和数据
- 提供商管理: 运行时、中间件、操作系统、基础设施

**3. 软件即服务（SaaS）:**
- 通过互联网提供的完整应用
- 示例: Office 365、Salesforce、Gmail、Dropbox
- 用户管理: 仅数据和使用

**1. 公共云:**
- 向公众提供的服务
