# Fundamentals of Data Communication
# 数据通信的基本知识

## 1. Fundamental Concepts
## 1. 基础概念

### 1.1 Definition and Components of Data Communication
### 1.1 数据通信的定义和组成要素

**Definition:** Data communication is the process of exchanging digital information between two or more devices through a transmission medium, following specific protocols and standards.

**定义：** 数据通信是指在两个或多个设备之间通过传输介质，遵循特定协议和标准来交换数字信息的过程。

**Five Key Components:**
**五个关键组成要素：**

1. **Message (Data)**: Information to be communicated (text, numbers, images, video, audio)
1. **消息（数据）**：要传输的信息（文本、数字、图像、视频、音频）

2. **Sender (Source)**: Device that generates and sends the data
2. **发送方（源）**：生成并发送数据的设备

3. **Receiver (Destination)**: Device that receives the data
3. **接收方（目的地）**：接收数据的设备

4. **Transmission Medium**: Physical path for data transmission
4. **传输介质**：数据传输的物理路径

5. **Protocol**: Set of rules governing data communication
5. **协议**：管理数据通信的规则集

**Basic Communication Model:**
**基本通信模型：**

```
┌──────────┐    ┌──────────┐    ┌─────────┐    ┌──────────┐    ┌──────────┐
│  Source  │───▶│ Transmit │───▶│ Channel │───▶│ Receiver │───▶│Destination│
│   源端   │    │   发送器  │    │  信道   │    │  接收器   │    │  目的端   │
└──────────┘    └──────────┘    └─────────┘    └──────────┘    └──────────┘
                                       │
                                       ▼
                                  ┌─────────┐
                                  │  Noise  │
                                  │  噪声   │
                                  └─────────┘
```

### 1.2 Classification of Communication Systems
### 1.2 通信系统的分类

**By Transmission Mode:**
**按传输模式分类：**

1. **Simplex**: One-way communication only (e.g., broadcast radio, TV)
1. **单工**：仅单向通信（如广播电台、电视）

2. **Half-Duplex**: Two-way communication, but not simultaneously (e.g., walkie-talkie)
2. **半双工**：双向通信，但不能同时（如对讲机）

3. **Full-Duplex**: Simultaneous two-way communication (e.g., telephone, Ethernet)
3. **全双工**：同时双向通信（如电话、以太网）

**By Signal Type:**
**按信号类型分类：**

1. **Analog Communication**: Continuous signal variations (e.g., traditional telephone)
1. **模拟通信**：连续的信号变化（如传统电话）

2. **Digital Communication**: Discrete signal levels (e.g., computer networks)
2. **数字通信**：离散的信号电平（如计算机网络）

**By Transmission Topology:**
**按传输拓扑分类：**

1. **Point-to-Point**: Direct connection between two devices
1. **点对点**：两个设备之间的直接连接

2. **Multipoint (Broadcast)**: One sender to multiple receivers
2. **多点（广播）**：一个发送方到多个接收方

### 1.3 Key Performance Metrics
### 1.3 关键性能指标

**1. Bandwidth (带宽)**

- **Definition**: Range of frequencies a channel can carry, or maximum data rate
- **定义**：信道能够传输的频率范围，或最大数据速率
- **Unit**: Hertz (Hz) for analog, bits per second (bps) for digital
- **单位**：模拟信号用赫兹（Hz），数字信号用比特每秒（bps）

```
Bandwidth (Hz) = f_high - f_low
带宽（Hz）= 最高频率 - 最低频率
```

**2. Throughput (吞吐量)**

- **Definition**: Actual data rate achieved in practice
- **定义**：实际中达到的数据传输速率
- **Relationship**: Throughput ≤ Bandwidth (due to overhead and errors)
- **关系**：吞吐量 ≤ 带宽（由于开销和错误）

**3. Latency (Delay) (延迟)**

```
Total Latency = Propagation Delay + Transmission Delay + Processing Delay + Queuing Delay
总延迟 = 传播延迟 + 传输延迟 + 处理延迟 + 排队延迟

Propagation Delay = Distance / Signal Propagation Speed
传播延迟 = 距离 / 信号传播速度

Transmission Delay = Data Size / Bandwidth
传输延迟 = 数据大小 / 带宽
```

**4. Bit Error Rate (BER) (误码率)**

```
BER = Number of Bit Errors / Total Number of Bits Transmitted
误码率 = 错误比特数 / 传输总比特数
```

**5. Jitter (抖动)**

- **Definition**: Variation in packet arrival time
- **定义**：数据包到达时间的变化
- **Impact**: Critical for real-time applications (voice, video)
- **影响**：对实时应用（语音、视频）至关重要

- Represented by sine wave: `s(t) = A·sin(2πft + φ)`

- Represented by square waves
- 用方波表示
- Advantages: noise immunity, easy processing, encryption
- 优点：抗噪声、易处理、加密

### 2.3 Channel Capacity
### 2.3 信道容量

#### 2.3.1 Nyquist Theorem (奈奎斯特定理)
For noiseless channel:
对于无噪声信道：

```
C = 2B × log₂(M)

Where:
C = Maximum data rate (bps) / 最大数据速率
B = Bandwidth (Hz) / 带宽
M = Number of signal levels / 信号电平数
```

**Example:**
**示例：**
```
Given: B = 3000 Hz, M = 4 (4 signal levels)
给定：B = 3000 Hz，M = 4（4个信号电平）

C = 2 × 3000 × log₂(4) = 2 × 3000 × 2 = 12,000 bps
```

#### 2.3.2 Shannon's Theorem (香农定理)
For noisy channel:
对于有噪声信道：

```
C = B × log₂(1 + SNR)

Where:
C = Maximum data rate (bps) / 最大数据速率
B = Bandwidth (Hz) / 带宽
SNR = Signal-to-Noise Ratio / 信噪比

SNR(dB) = 10 × log₁₀(Signal Power / Noise Power)
信噪比(dB) = 10 × log₁₀(信号功率 / 噪声功率)
```

**Example:**
**示例：**
```
Given: B = 3000 Hz, SNR = 30 dB
给定：B = 3000 Hz，信噪比 = 30 dB

SNR = 10^(30/10) = 1000
C = 3000 × log₂(1 + 1000) ≈ 3000 × 9.97 ≈ 29,900 bps
```

**Key Insight:**
**关键见解：**
- Nyquist: Theoretical limit without noise
- 奈奎斯特：无噪声理论极限
- Shannon: Practical limit considering noise
- 香农：考虑噪声的实际极限
- Increasing bandwidth or SNR improves capacity
- 增加带宽或信噪比可提高容量

### 2.5 Modulation and Demodulation
### 2.5 调制与解调

**Purpose of Modulation:**
**调制目的：**
- Convert signal to suitable frequency range
- 将信号转换到合适的频率范围
- Enable multiplexing
- 实现多路复用
- Improve noise immunity
- 提高抗噪声能力
- Reduce antenna size
- 减小天线尺寸

#### 2.5.1 Analog Modulation
#### 2.5.1 模拟调制

**1. Amplitude Modulation (AM) - 幅度调制**

```
s(t) = [A + m(t)] × cos(2πf_c·t)

Where:
m(t) = Message signal / 消息信号
f_c = Carrier frequency / 载波频率
A = Carrier amplitude / 载波幅度
```

**2. Frequency Modulation (FM) - 频率调制**

```
s(t) = A × cos(2πf_c·t + 2πk_f∫m(t)dt)

Frequency varies with message amplitude
频率随消息幅度变化
```

**3. Phase Modulation (PM) - 相位调制**

```
s(t) = A × cos(2πf_c·t + k_p·m(t))

Phase varies with message amplitude
相位随消息幅度变化
```

#### 2.5.2 Digital Modulation
#### 2.5.2 数字调制

**1. Amplitude Shift Keying (ASK) - 幅移键控**

- Binary 1: High amplitude
- 二进制1：高幅度
- Binary 0: Low amplitude (or zero)
- 二进制0：低幅度（或零）
- Simple but susceptible to noise
- 简单但易受噪声影响

**2. Frequency Shift Keying (FSK) - 频移键控**

- Binary 1: Frequency f₁
- 二进制1：频率f₁
- Binary 0: Frequency f₀
- 二进制0：频率f₀
- Better noise immunity than ASK
- 比ASK有更好的抗噪声能力
- Used in: Modems, RFID
- 应用于：调制解调器、RFID

**3. Phase Shift Keying (PSK) - 相移键控**

**Binary PSK (BPSK):**
**二进制PSK（BPSK）：**
- Binary 1: Phase 0°
- 二进制1：相位0°
- Binary 0: Phase 180°
- 二进制0：相位180°

**Quadrature PSK (QPSK):**
**正交PSK（QPSK）：**
- 4 phase states (0°, 90°, 180°, 270°)
- 4个相位状态（0°、90°、180°、270°）
- Transmits 2 bits per symbol
- 每个符号传输2比特
- Higher spectral efficiency
- 更高的频谱效率

**4. Quadrature Amplitude Modulation (QAM) - 正交幅度调制**

- Combines ASK and PSK
- 结合ASK和PSK
- Multiple amplitude and phase combinations
- 多种幅度和相位组合
- High spectral efficiency
- 高频谱效率
- Common variants: 16-QAM, 64-QAM, 256-QAM
- 常见变体：16-QAM、64-QAM、256-QAM
- Used in: WiFi, cable modems, LTE/5G
- 应用于：WiFi、电缆调制解调器、LTE/5G

## 3. Data Encoding and Transmission
## 3. 数据编码与传输

### 3.1 Digital Data Encoding
### 3.1 数字数据编码

#### 3.1.1 Line Coding Schemes
#### 3.1.1 线路编码方案

**1. Non-Return to Zero (NRZ) - 不归零编码**

**NRZ-Level (NRZ-L):**
**NRZ电平：**
- Binary 1: Positive voltage
- 二进制1：正电压
- Binary 0: Negative voltage
- 二进制0：负电压
- Problem: Long strings of same bit cause synchronization issues
- 问题：相同比特的长串导致同步问题

**NRZ-Inverted (NRZ-I):**
**NRZ反转：**
- Binary 1: Transition at bit boundary
- 二进制1：位边界转换
- Binary 0: No transition
- 二进制0：无转换
- Better for synchronization
- 更好的同步性

**2. Manchester Encoding - 曼彻斯特编码**

- Transition at middle of each bit period
- 每个位周期中间转换
- Binary 1: Low-to-high transition
- 二进制1：低到高转换
- Binary 0: High-to-low transition
- 二进制0：高到低转换
- Advantages: Self-clocking, no DC component
- 优点：自时钟、无直流分量
- Disadvantage: Requires 2× bandwidth
- 缺点：需要2倍带宽
- Used in: Ethernet (10BASE-T)
- 应用于：以太网（10BASE-T）

**3. Differential Manchester - 差分曼彻斯特编码**

- Always transition at middle of bit period
- 总是在位周期中间转换
- Binary 1: No transition at start
- 二进制1：起始无转换
- Binary 0: Transition at start
- 二进制0：起始转换
- Better noise immunity than Manchester
- 比曼彻斯特有更好的抗噪声能力
- Used in: Token Ring
- 应用于：令牌环网

**4. 4B/5B Encoding - 4B/5B编码**

- Maps 4 data bits to 5 code bits
- 将4个数据位映射到5个编码位
- Ensures sufficient transitions for synchronization
- 确保足够的转换以实现同步
- Efficiency: 80% (vs 50% for Manchester)
- 效率：80%（曼彻斯特为50%）
- Used in: 100BASE-TX Fast Ethernet
- 应用于：100BASE-TX快速以太网

### 3.2 Analog-to-Digital Conversion (PCM)
### 3.2 模拟到数字转换（PCM）

**Pulse Code Modulation (PCM) Process:**
**脉冲编码调制（PCM）过程：**

**1. Sampling (采样)**

**Nyquist Sampling Theorem:**
**奈奎斯特采样定理：**

```
f_s ≥ 2 × f_max

Where:
f_s = Sampling frequency / 采样频率
f_max = Maximum frequency in signal / 信号最高频率
```

**Example:**
**示例：**
```
Voice signal: f_max = 4 kHz
语音信号：最高频率 = 4 kHz

Minimum sampling rate: f_s = 2 × 4000 = 8000 Hz
最小采样率：f_s = 2 × 4000 = 8000 Hz

Standard: 8000 samples/second (telephone systems)
标准：每秒8000个样本（电话系统）
```

**2. Quantization (量化)**

- Convert continuous amplitude to discrete levels
- 将连续幅度转换为离散电平
- Number of levels: L = 2ⁿ (n = number of bits)
- 电平数：L = 2ⁿ（n = 位数）

**3. Encoding (编码)**

- Assign binary code to each quantized level
- 为每个量化电平分配二进制码
- Example: 8-bit encoding → 256 levels
- 示例：8位编码 → 256个电平

**PCM Bit Rate Calculation:**
**PCM比特率计算：**

```
Bit Rate = Sampling Rate × Bits per Sample × Number of Channels
比特率 = 采样率 × 每样本位数 × 通道数

Example (CD Audio):
示例（CD音频）：

Bit Rate = 44,100 Hz × 16 bits × 2 channels = 1,411,200 bps ≈ 1.4 Mbps
```

### 3.3 Data Compression
### 3.3 数据压缩

**Lossless Compression (无损压缩)**

- Original data can be perfectly reconstructed
- 可以完美重建原始数据
- Examples: ZIP, GZIP, LZW, Huffman coding
- 示例：ZIP、GZIP、LZW、哈夫曼编码
- Compression ratio: 2:1 to 3:1 typically

### 4.2 Error Detection Techniques
### 4.2 差错检测技术

#### 4.2.1 Parity Check
#### 4.2.1 奇偶校验

**Simple Parity (单维奇偶校验)**

- Add one parity bit to data
- 向数据添加一个奇偶校验位
- **Even parity**: Total 1s is even
- **偶校验**：总1数为偶数
- **Odd parity**: Total 1s is odd
- **奇校验**：总1数为奇数

```
Example (Even Parity):
示例（偶校验）：

Data: 1011001 (4 ones, even)
数据：1011001（4个1，偶数）
Parity bit: 0
校验位：0
Transmitted: 10110010
传输：10110010

If received: 10110110 (5 ones, odd) → Error detected
如果接收：10110110（5个1，奇数）→ 检测到错误
```

**Limitations:**
**局限性：**
- Cannot detect even number of errors
- 不能检测偶数个错误
- Cannot correct errors
- 不能纠正错误

#### 4.2.2 Cyclic Redundancy Check (CRC)
#### 4.2.2 循环冗余校验（CRC）

**Most powerful error detection method**
**最强大的错误检测方法**

**Common CRC Polynomials:**
**常用CRC多项式：**

| Name      | Polynomial        | Bits | Applications    |

| CRC-CCITT | x¹⁶+x¹²+x⁵+1      | 16   | HDLC, Bluetooth |

| 名称      | 多项式            | 位数 | 应用        |
| --------- | ----------------- | ---- | ----------- |

**CRC Capabilities:**
**CRC能力：**

#### 4.2.3 校验和

### 4.3 Error Correction Techniques
### 4.3 差错纠正技术

#### 4.3.1 Hamming Code
#### 4.3.1 海明码

**Forward Error Correction (FEC)**
**前向纠错（FEC）**

- Can detect and correct single-bit errors
- 可以检测和纠正单比特错误
- Can detect (but not correct) two-bit errors
- 可以检测（但不能纠正）双比特错误

**Hamming Distance:**
**海明距离：**

```
Hamming Distance = Number of bit positions where two codewords differ
海明距离 = 两个码字不同位位置的数量

For error detection: d_min ≥ s + 1
对于错误检测：d_min ≥ s + 1

For error correction: d_min ≥ 2t + 1
对于错误纠正：d_min ≥ 2t + 1

Where:
d_min = Minimum Hamming distance / 最小海明距离
s = Number of errors to detect / 要检测的错误数
t = Number of errors to correct / 要纠正的错误数
```

#### 4.3.2 Advanced Error Correction
#### 4.3.2 高级纠错码

- Iterative decoding
- 迭代解码
- Used in: 3G, 4G, satellite

**LDPC (Low-Density Parity-Check) Codes**

### 4.4 Automatic Repeat Request (ARQ)
### 4.4 自动重传请求（ARQ）

#### 4.4.1 Stop-and-Wait ARQ
#### 4.4.1 停止-等待ARQ

**Operation:**
**操作：**

1. Sender transmits one frame
1. 发送方传输一帧
2. Wait for acknowledgment (ACK) or negative ACK (NAK)
2. 等待确认（ACK）或否定确认（NAK）
3. If ACK received, send next frame
3. 如果收到ACK，发送下一帧
4. If NAK or timeout, retransmit
4. 如果NAK或超时，重传

**Disadvantage:** Low efficiency for high latency links
**缺点：** 高延迟链路效率低

#### 4.4.2 Go-Back-N ARQ
#### 4.4.2 回退N帧ARQ

**Features:**
**特点：**

- Sender can transmit multiple frames (window size N)
- 发送方可以传输多帧（窗口大小N）
- Receiver only accepts frames in order
- 接收方只按顺序接受帧
- If error detected, reject that frame and all subsequent frames
- 如果检测到错误，拒绝该帧及所有后续帧
- Sender retransmits from error frame
- 发送方从错误帧重传

**Advantage:** Better efficiency than Stop-and-Wait
**优点：** 比停止-等待效率更好

**Disadvantage:** May retransmit many correct frames
**缺点：** 可能重传许多正确的帧

#### 4.4.3 Selective Repeat ARQ
#### 4.4.3 选择重传ARQ

**Features:**
**特点：**

- Both sender and receiver have windows
- 发送方和接收方都有窗口
- Receiver accepts out-of-order frames
- 接收方接受乱序帧
- Only retransmit frames with errors
- 只重传有错误的帧
- Most efficient ARQ protocol
- 最高效的ARQ协议

**Advantage:** Highest efficiency
**优点：** 最高效率

**Disadvantage:** More complex implementation
**缺点：** 实现更复杂

**ARQ Comparison:**
**ARQ比较：**

| Protocol         | Sender Window | Receiver Window | Efficiency | Complexity |
| ---------------- | ------------- | --------------- | ---------- | ---------- |
| Stop-and-Wait    | 1             | 1               | Low        | Low        |
| Go-Back-N        | N             | 1               | Medium     | Medium     |
| Selective Repeat | N             | N               | High       | High       |

| 协议      | 发送方窗口 | 接收方窗口 | 效率 | 复杂度 |
| --------- | ---------- | ---------- | ---- | ------ |
| 停止-等待 | 1          | 1          | 低   | 低     |
| 回退N帧   | N          | 1          | 中   | 中     |

**Algorithm:**
**算法：**

```

3. 传输时侦听冲突

4. If collision detected:
4. 如果检测到冲突：
   a. Stop transmission
   a. 停止传输
   
   b. Send jam signal
   b. 发送干扰信号
   
   c. Wait random backoff time
   c. 等待随机退避时间
   
   d. Retry
   d. 重试
```

**CSMA/CA (Collision Avoidance) - 冲突避免**

- 事先尝试避免冲突

### 6.4 Reliable Transmission Mechanisms
### 6.4 可靠传输机制

**TCP Reliability Features:**
**TCP可靠性特性：**

**1. Sequence Numbers (序列号)**
- Track data bytes
- 跟踪数据字节
- Detect missing or duplicate segments
- 检测丢失或重复段

**2. Acknowledgments (确认)**
- Confirm receipt
- 确认接收
- Cumulative or selective ACK
- 累积或选择确认

**3. Retransmission Timeout (重传超时)**
```
RTO = EstimatedRTT + 4 × DevRTT

Where:
RTO = Retransmission Timeout / 重传超时
EstimatedRTT = Average round-trip time / 平均往返时间
DevRTT = RTT deviation / RTT偏差
```

**4. Fast Retransmit (快速重传)**
- Retransmit after 3 duplicate ACKs
- 3个重复ACK后重传
- Don't wait for timeout
- 不等待超时

**5. Flow Control (流量控制)**
- Sliding window protocol
- 滑动窗口协议
- Receiver advertises window size
- 接收方通告窗口大小
- Prevents buffer overflow
- 防止缓冲区溢出

**6. Congestion Control (拥塞控制)**

**Algorithms:**
**算法：**
- **Slow Start**: Exponential growth
- **慢启动**：指数增长
- **Congestion Avoidance**: Linear growth
- **拥塞避免**：线性增长
- **Fast Recovery**: After packet loss
- **快速恢复**：数据包丢失后

4. **多路访问**：根据网络拓扑和需求选择正确的MAC协议（CSMA/CD、CSMA/CA）

5. **Layered Architecture**: Leverage OSI and TCP/IP models for systematic network design and troubleshooting
5. **分层架构**：利用OSI和TCP/IP模型进行系统化网络设计和故障排除
